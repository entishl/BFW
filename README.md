# Brain-Firewall (BFW) 🧠🛡️

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

> BFW（Brain-Firewall，你也可以理解为 Bilibili-Firewall）是一款基于大语言模型（LLM）的 Manifest V3 浏览器扩展，旨在帮助用户重新获得数字生活的主动权。

BFW 通过分析网页内容并结合用户自定义的分类规则，在你不经意间陷入高情绪消耗、低价值内容时，进行温和的干预与劝退。

## ✨ 核心特性

- **🤖 智能内容分析**：自动抓取当前页面的关键信息，如视频标题、UP主、标签、简介和~~热门评论~~（B站评论是懒加载，要抓影响效率，当前写了逻辑但是实际上抓不到，不过仅凭标题、简介、标签已经基本足够AI判断了，~~顺便省了token~~），并调用你配置的 LLM（支持 OpenAI 兼容接口，如 OpenAI、DeepSeek、Ollama 等）进行智能分类判定。
- **🛡️ 温和干预（放行优先）**：只要 AI 判定的分类中，存在任何一个“未被用户勾选拦截”的分类，则整体放行。只有当内容符合你设定的拦截标准时，才会触发全屏毛玻璃遮罩并强制暂停视频。
- **✌️ 尊重用户选择**：只做“温和提醒”而非“强制管控”。你始终拥有点击【我知道风险，但我今天想放纵一下】以继续观看的权利。
- **⚡ 无感交互体验**：在后台分析期间，页面正常交互、视频正常播放，仅展示轻量级 Toast 提示。完善的 SPA 路由监听确保在无刷新的页面跳转中依然能精确判定。
- **🔒 隐私与安全**：你的 API Key 仅保存在浏览器本地缓存中，绝不上传任何第三方服务；只申请最少的浏览器权限。

## 🎯 当前支持平台（MVP 阶段）

- **Bilibili (B站) 视频播放页** (`*://*.bilibili.com/video/BV*`)
- **暂时不支持**：
    - 番剧页
    - 直播页
    - 动态页

*（技术架构已模块化，可扩展到其他平台。但是作者一般只在b站上浪费时间和情绪，所以不一定做，欢迎提PR,欢迎fork了自己做）*

## 🚀 安装与配置

1. 下载本项目的代码。
2. 打开 Chrome / Edge 浏览器的扩展管理页面（`chrome://extensions/` 或 `edge://extensions/`）。
3. 开启右上角的 **开发者模式**。
4. 点击 **加载已解压的扩展程序**，选择本项目的根目录。
5. 点击扩展图标进入 **设置 (Options)** 页面，进行如下配置：
   - **API 配置**：填入兼容 OpenAI 格式的 `Base URL`、`API Key` 和 `Model` 名称。
   - **分类规则管理**：动态添加或修改你想拦截或放行的内容分类（默认提供：有害、无用、技术、学习、娱乐等示例）。
     > 请注意，如果一个页面被AI识别为多个分类，只要存在任意一个分类是你**允许放行**的，则会放行，反之则拦截。

     > 比如，你不想在任何“无用”的娱乐圈信息上浪费时间，但是你特别喜欢周杰伦，就想看周杰伦。那么你可以增加一个`周杰伦`的分类进行放行。
   - **AI 语气设置**：自定义 AI 劝退你时的语气（例如：“像一个温柔心理医生一样劝我”）。

## 🏗️ 架构设计

本项目基于原生 HTML / CSS / JavaScript 开发，无复杂框架依赖，结构清晰：

- `manifest.json`: 声明扩展配置、权限、后台 Worker 和 Content Scripts。
- `content_script.js`: 负责 URL 监听、SPA 路由监听、页面 DOM 数据抓取以及 Toast 和遮罩层 UI 的注入与视频控制。
- `background.js`: 扩展的大脑，负责统一的 API 请求处理、动态 Prompt 组装、超时与异常控制、以及业务放行/拦截逻辑的统一决策。
- `options.html` / `options.js`: 用户设置中心。
- `platform/bilibili.js`: B站专属的 DOM 解析与数据提取逻辑模块（方便未来横向扩展其他平台）。

> **调试提示**：
> 在开发模式下，可以打开浏览器开发者工具查看 `console.log` 输出的执行日志、DOM 抓取数据及 AI 的原始返回结果。生产模式默认关闭冗余日志。
